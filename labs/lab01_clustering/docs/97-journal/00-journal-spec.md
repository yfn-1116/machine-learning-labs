# Lab01 实验日志规范（Spec）

目标：记录真实发生的实验/学习/研发进展，便于复盘与复现。
不强制每日填写，只记录真实发生工作的日期。

## 组织方式
- 按周组织：Week X（YYYY-MM-DD ~ YYYY-MM-DD）
- 每周必须包含：
  - 本周学习目标
  - 每日学习日志（多天条目）
  - 本周总结与反思
  - 下周学习计划

## 内容要求
- 保留所有原始技术细节：命令、参数、代码块、输出、错误信息、路径
- 不要为了“好看”删除细节；细节是复现的关键
- 如果某个结论不确定，用 TODO 标注，不要编造

## 每日固定 5 个小标题（必须）
- 学习内容
- 关键收获
- 遇到的问题
- 解决方案
- 反思与下一步

### 遇到的问题
- Windows 与 WSL 同名仓库导致 IDE 看不到新建目录（路径不一致）。
- 使用 `cat > ... <<'EOF'` 写文件时目录不存在或被中断，导致文件未落盘。
- docs 结构调整过程中容易误删/遗漏，需要安全迁移流程。

### 解决方案
- 统一在 WSL 文件系统中开发，IDE 打开 `\\wsl.localhost\Ubuntu\home\yfn\repos\...`。
- 写文件前先 `mkdir -p`，避免 `No such file or directory`。
- 迁移采用“先复制→tree 验证→再 rm -rf 清理旧目录”的流程确保安全。
