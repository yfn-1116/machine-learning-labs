# 机器学习实验 8：KNN 与 ID3 决策树算法实验

## 实验概述

本实验为机器学习系列实验的第八部分，重点在于掌握监督学习中的 K 近邻（K-Nearest Neighbors, KNN）和 ID3 决策树两种经典分类算法。实验围绕直播电商用户意向预测场景展开，根据用户的行为特征（停留时长、小黄车点击次数、互动次数）判断用户是否为高意向下单用户。实验内容包括数据生成与预处理、自定义算法实现（不调用 sklearn 的高层封装）、模型训练与评估、混淆矩阵可视化分析以及冷启动场景性能对比等多个环节。

实验采用模拟生成的直播电商用户行为数据集（1000 个样本），通过 KNN（K=5，欧氏距离，Z-Score 标准化）和 ID3 决策树（无剪枝，特征分桶离散化）两种算法进行对比实验。实验最终生成 4 张可视化图表，从数据分布、混淆矩阵、冷启动准确率对比等多个角度展示两种分类算法的表现差异。

## 实验目标

1. **理解 KNN 算法原理**：掌握基于距离的懒惰学习方法，理解 K 值选择、距离度量、多数投票等核心概念。
2. **实现原生 KNN 分类器**：不使用 sklearn 封装，手动实现欧氏距离计算、K 近邻查找、多数投票分类的完整流程。
3. **理解 ID3 决策树算法原理**：掌握信息熵、信息增益、递归树构建等核心概念。
4. **实现原生 ID3 决策树**：手动实现熵计算、信息增益计算、递归建树、基于树的预测推理。
5. **处理连续特征的离散化**：针对 ID3 只能处理离散特征的特点，实现等宽分桶策略将连续特征转为类别特征。
6. **评估与对比模型性能**：使用准确率、精确率、召回率、F1 值和混淆矩阵量化评估两个模型。
7. **冷启动场景分析**：在不同训练集比例（10%、30%、50%、70%）下对比 KNN 和 ID3 的准确率变化，分析算法在小样本场景下的表现。

### 数据集

实验采用模拟生成的直播电商用户行为数据集，包含 1000 个样本，每个样本有 3 个特征和 1 个二分类标签：

| 特征列 | 数据类型 | 取值范围 | 描述 |
|--------|---------|---------|------|
| 停留时长 | float | [1, 300] | 用户在直播间的停留时间（秒），服从 Gamma 分布 |
| 小黄车点击次数 | int | [0, 20] | 用户点击购物车（小黄车）的次数，服从 Poisson 分布 |
| 互动次数 | int | [0, 30] | 用户点赞、评论等互动次数，服从 Poisson 分布 |
| 标签 | int | {0, 1} | 0=非高意向用户，1=高意向下单用户 |

标签生成公式：

```
score = 0.035 × 停留时长 + 0.9 × 小黄车点击次数 + 0.45 × 互动次数 + N(0, 2.2)
标签 = (score ≥ 7.5).astype(int)
```

特征之间存在关联性：停留时长越长，点击购物车和互动的期望次数越高，符合真实直播电商场景的行为模式。

## 方法

### KNN 算法

**核心原理**：K 近邻（K-Nearest Neighbors）是一种基于实例的懒惰学习方法。它不显式学习一个判别函数，而是将训练数据直接存储起来，在预测时通过计算测试样本与所有训练样本的距离，找到距离最近的 K 个邻居，然后通过多数投票确定测试样本的类别。

**本实验实现要点**：

1. **距离度量**：使用欧氏距离（Euclidean Distance）衡量样本间相似度：
   $$d(x_1, x_2) = \sqrt{\sum_{i=1}^{n}(x_{1i} - x_{2i})^2}$$

2. **K 值选择**：K=5，通过 5 个最近邻投票决定类别归属。

3. **特征标准化**：使用 Z-Score 标准化（均值为 0，标准差为 1）消除特征尺度差异对距离计算的影响。标准化参数仅从训练集计算，测试集使用相同的均值和标准差进行变换，避免数据泄露。

4. **预测策略**：对距离最近的 K 个邻居的标签进行计数，返回出现次数最多的类别。

### ID3 决策树算法

**核心原理**：ID3（Iterative Dichotomiser 3）是一种基于信息增益的决策树学习算法。它通过递归地选择信息增益最大的特征作为分裂节点，将数据集划分为纯度更高的子集，最终构建一棵可用于分类的决策树。

**本实验实现要点**：

1. **信息熵**：衡量数据集纯度的指标：
   $$H(Y) = -\sum_{i} p_i \log_2 p_i$$

2. **信息增益**：衡量某个特征对数据集纯度的提升程度：
   $$IG(特征) = H(Y) - \sum_{v} \frac{|S_v|}{|S|} H(S_v)$$

3. **递归建树**：
   - 终止条件 1：当前节点所有样本属于同一类别 → 返回该类别作为叶子节点
   - 终止条件 2：无可选特征或样本数小于 `min_samples_split=10` → 返回多数类别
   - 迭代步骤：计算所有特征的信息增益 → 选择增益最大的特征 → 按特征值划分子集 → 递归建树

4. **连续特征离散化**：ID3 要求特征为离散类型，因此对三个连续特征进行分桶处理：

   | 原始特征 | 分桶规则 | 离散取值 |
   |---------|---------|---------|
   | 停留时长 | <30→短停留，<60→中停留，≥60→长停留 | 短停留/中停留/长停留 |
   | 小黄车点击次数 | 0→无，≤3→低，≤7→中，>7→高 | 无/低/中/高 |
   | 互动次数 | 同点击次数分桶规则 | 无/低/中/高 |

5. **预测推理**：从根节点开始，根据样本的特征值递归向下查找，到达叶子节点后返回对应的类别标签。

### 冷启动实验设计

为模拟实际应用中标注数据稀缺的场景，实验设计了冷启动对比分析：将原始训练集按 10%、30%、50%、70% 的比例进行子采样，分别训练 KNN 和 ID3 模型，并在完整的测试集上评估准确率，观察两种算法在不同数据量下的表现差异。

**ID3 决策树结构**（以字典形式表示）：

```
小黄车点击次数分桶
├── 低 → 互动次数分桶
│   ├── 低 → 停留时长分桶 → 全部分类为 0
│   ├── 中 → 停留时长分桶 → 全部分类为 0
│   ├── 高 → 停留时长分桶 → 长停留分类为 1，其余分类为 0
│   └── 无 → 0
├── 无 → 互动次数分桶
│   ├── 低 → 0
│   ├── 中 → 停留时长分桶 → 0
│   ├── 高 → 0
│   └── 无 → 0
├── 中 → 停留时长分桶
│   ├── 长停留 → 互动次数分桶 → 低/中/高→1，无→0
│   └── 中停留 → 0
└── 高 → 1
```

从树结构可以看出：
- 小黄车点击次数是最重要的分裂特征（根节点），这与标签生成公式中该特征的权重最高（0.9）一致。
- 当小黄车点击次数为"高"（>7 次）时，直接判定为高意向用户。
- 当点击次数为"中"且停留时间为"长停留"时，进一步根据互动次数判断。
- 当点击次数为"低"或"无"时，需要结合互动次数和停留时长进一步细分。

## 结果分析与讨论

### 算法性能分析

1. **准确率与 F1 值**：KNN 的综合表现优于 ID3。KNN 的准确率为 86.00%，F1 值为 0.6500；ID3 的准确率为 83.33%，F1 值为 0.4681。KNN 在召回率（58.21%）上大幅领先 ID3（32.84%），说明 KNN 能更有效地捕捉高意向用户。

2. **精确率 vs 召回率权衡**：ID3 具有更高的精确率（81.48% vs 73.58%），意味着 ID3 预测为高意向用户时更加可靠（假阳性更少）。但 ID3 的召回率较低（32.84%），大量高意向用户被遗漏（假阴性较多）。相比之下，KNN 在召回率上有明显优势，能识别出更多真正的高意向用户，但以更多的误报为代价。

3. **混淆矩阵分析**：
   - KNN 的混淆矩阵显示，在 300 个测试样本中，KNN 正确分类了 258 个样本。
   - ID3 的混淆矩阵显示，ID3 正确分类了 250 个样本，但其中真正的高意向用户仅识别出约 22 个，漏掉了约 45 个。

4. **时间效率**：
   - KNN 的训练几乎是瞬时的（0.00013 秒），因为训练过程只是存储数据。
   - KNN 的预测时间较长（0.46553 秒），因为需要计算测试样本与所有训练样本的距离。
   - ID3 的训练时间稍长（0.00894 秒），因为需要递归计算信息增益。
   - ID3 的预测非常快（0.00250 秒），只需在树结构中做条件判断。

### 冷启动场景分析

1. **KNN 整体优势**：在所有训练集比例下，KNN 的准确率均高于 ID3，体现了 KNN 作为非参数方法在小样本场景下的稳定性。

2. **KNN 随数据量增长**：KNN 的准确率随训练数据增加而稳定提升（从 85.00% 到 89.00%），符合统计学习理论中更多数据带来更好性能的预期。

3. **ID3 表现平缓**：ID3 的准确率在不同数据量下变化不大（约 83%），甚至在 70% 训练数据时出现轻微下降（83.33%）。这表明 ID3 的离散化分桶策略可能损失了连续特征中的细粒度信息，限制了模型从更多数据中获益的能力。

4. **极端小样本场景**：即使仅有 10% 的训练数据（70 个样本），KNN 仍能达到 85% 的准确率，表明数据本身具有较强的可分离性。

### 算法对比总结

| 维度 | KNN | ID3 |
|------|-----|-----|
| **模型类型** | 懒惰学习（基于实例） | 急切学习（构建全局模型） |
| **特征要求** | 连续数值特征 | 离散类别特征 |
| **可解释性** | 低（黑盒投票） | 高（清晰的树结构规则） |
| **小样本表现** | 优秀 | 一般 |
| **预测效率** | 慢（需遍历全部训练数据） | 极快（树查找 O(logN)） |
| **训练效率** | 极快（仅存储数据） | 较慢（递归建树） |
| **连续特征处理** | 天然支持（需标准化） | 需要离散化 |
| **本实验综合表现** | **更优** | 受限于离散化信息损失 |

## 结论

### 实验成果

本实验成功实现了 KNN 和 ID3 两种经典分类算法的完整流程，达成以下目标：

1. **算法原生实现**：不依赖 sklearn 的高层封装，手动实现了 KNN（欧氏距离计算、K 近邻查找、多数投票）和 ID3（信息熵、信息增益、递归树构建、树推理）两个算法。
2. **数据预处理**：完成了针对 KNN 的 Z-Score 标准化和针对 ID3 的连续特征分桶离散化。
3. **模型评估**：使用准确率、精确率、召回率、F1 值和混淆矩阵从多维度评估模型性能。
4. **冷启动分析**：在 4 种不同训练集比例下对比算法表现，评估小样本场景的适用性。
5. **可视化呈现**：生成了数据分布散点图、混淆矩阵热力图、冷启动准确率对比折线图等多张图表辅助分析。

### 核心发现

1. **KNN 综合表现更优**：在直播电商用户意向预测任务中，KNN 以 86.00% 的准确率和 0.65 的 F1 值优于 ID3（83.33% 准确率、0.47 F1 值），尤其在召回率上有明显优势。
2. **特征离散化带来的信息损失**：ID3 受限于离散化策略，连续特征分桶后丢失了细粒度信息，导致模型无法充分利用数据量增加带来的性能提升。
3. **算法选择需结合应用场景**：如果需要高可解释性（如业务规则审计），ID3 的树结构具有天然优势；如果更关注预测性能且对推理速度要求不高，KNN 是更好的选择。
4. **小样本场景 KNN 更稳健**：在仅有 10% 训练数据时，KNN 仍能达到 85% 的准确率，适合标注数据稀缺的冷启动场景。

### 改进方向

1. **KNN 优化**：引入加权投票（距离越近权重越大）、KD-Tree 或 Ball-Tree 加速近邻查找、通过交叉验证自动选择最优 K 值。
2. **ID3 增强**：实现连续特征的二分法分裂（如 C4.5 算法）避免信息损失、引入剪枝策略防止过拟合、支持缺失值处理。
3. **特征工程**：探索更多用户行为特征（如进入直播间的来源渠道、关注与否、历史购买记录），提升模型表现。
4. **扩展对比**：引入更多分类算法（如逻辑回归、支持向量机、随机森林）进行综合对比。
